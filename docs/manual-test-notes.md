# Manual Test Notes

Use this checklist after making changes.

## Dashboard

- Start the app.
- Confirm the dashboard loads.
- Confirm failed jobs and stale jobs display.
- Confirm open incidents display.
- Confirm recent health summary display.

## Sites

- Open Sites.
- Add a sample site.
- Confirm the site appears in the table.

## Jobs

- Open Jobs.
- Add a failed sample job.
- Return to the Dashboard.
- Confirm the failed job count changes.

## Incidents

- Open Incidents.
- Create a new incident.
- Open the incident details page.
- Add a timeline note.
- Resolve the incident.

## Audit

- Open Audit.
- Confirm incident actions were recorded.

## Exports/API

- Open Exports/API.
- Open the JSON endpoints.
- Download the CSV exports.

## Tests

Run dotnet test from the repository root.
